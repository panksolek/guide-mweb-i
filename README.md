# guide-mweb-i

## 项目简介

`guide-mweb-i` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面以静态文件形式存放，不绑定特定域名或网站，旨在提供一种轻量、可访问的页面存档方式。

本仓库内容适合存放需要长期保留的页面快照、文档片段、演示页面或临时性 HTML 资源，便于通过 GitHub Pages 或本地浏览直接访问。

## 目录结构

```
guide-mweb-i/
├── pages/          # 存放主要 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── index.html      # 仓库根目录的入口页面（可选）
└── README.md       # 本说明文件
```

- `pages/`：每个 HTML 文件对应一个独立页面，文件名建议语义化。
- `assets/`：按需组织子目录，保持资源与页面的引用关系清晰。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，可直接在浏览器中打开。
- 页面内容不应当包含指向特定网站的链接或依赖外部服务。
- 归档页面应尽量自包含，避免使用外部资源导致失效。
- 若需引用图片或样式，建议将资源一并放入 `assets/` 目录中。

## 维护说明

- 本仓库由维护者不定期更新，新增页面请放入 `pages/` 目录并更新 `index.html` 入口（如有）。
- 不建议删除已有页面，以免破坏外部引用或存档完整性。
- 欢迎提交 Pull Request 添加新的独立页面，但请确保内容不违反平台政策。
- 如有问题或建议，请通过 Issues 进行讨论。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在本地打开 `pages/` 下的 HTML 文件，或通过 GitHub Pages 在线访问。
3. 如需部署，可将仓库设置为 GitHub Pages 源（选择 `main` 分支或 `/docs` 目录）。
